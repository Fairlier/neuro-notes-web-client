# Клиентское веб-приложение информационной системы NeuroNotes 

В данном репозитории представлен исходный код клиентского веб-приложения информационной системы интеллектуальной обработки аудио-текстовых данных, созданной в ходе выполнения выпускной квалификационной работы. Исходный код серверного приложения представлен в связанном репозитории [neuro-notes](https://github.com/Fairlier/neuro-notes). Приложение предоставляет графический интерфейс для управления пользовательскими записями, контроля процессов машинного обучения и настройки системных параметров. Функционал включает создание аудиозаметок со встроенной визуализацией частотного спектра, редактирование текстов с поддержкой разметки Markdown, семантический поиск и работу с изолированными диалоговыми панелями для общения с базой знаний пользователя. 

Архитектура клиентского приложения реализована по модели одностраничного веб-приложения с применением языка строгой статической типизации TypeScript. Для построения пользовательского интерфейса использована библиотека React 19. Управление структурой проекта опирается на принципы методологии функционально-сегментированного проектирования. Маршрутизация организована средствами библиотеки React Router. Управление серверным состоянием, кэшированием запросов и синхронизацией данных реализовано с помощью TanStack Query. Стилизация компонентов выполнена на базе фреймворка Tailwind CSS с интеграцией интерфейсных примитивов Radix UI. 

Пользовательский интерфейс адаптирован для управления гибридной интеллектуальной обработкой. В разделе настроек профиля реализован механизм конфигурации используемых моделей машинного обучения для каждой отдельной задачи: транскрибации, структурирования, резюмирования и диалогового поиска. Предусмотрена возможность задания пользовательских системных инструкций (промптов), управляющих поведением генеративных сетей. Взаимодействие с контекстом организовано через два режима диалогового поиска: по изолированному контексту отдельной заметки и по глобальному контексту всей совокупности пользовательских записей с применением технологий векторного поиска.

## Развертывание и запуск

Развертывание клиентского приложения возможно как из исходного кода репозитория, так и в составе объединённого релиза всей системы.

**Развертывание и запуск из исходного кода.**  
Для запуска клиентского приложения из исходного кода необходимо предварительно создать файл переменных окружения `.env` в корневой директории проекта на основе шаблона `.env.example`. Перед запуском требуется внимательно проверить и при необходимости изменить основные параметры конфигурации, прежде всего порт публикации клиентского приложения и адрес серверного API, задаваемый переменной `VITE_API_URL`. При использовании контейнерного сценария пользователь не должен изменять файл `docker-compose.yml`, так как все настраиваемые параметры выносятся в файл `.env`. После подготовки конфигурации запуск приложения выполняется в корневой директории проекта командой:

```bash
docker compose up -d
```

**Развертывание и запуск в составе объединённого релиза.**  
Для запуска готовой сборки всей системы в разделе релизов предоставляются объединённые файлы `docker-compose.yml` и `.env`. Данный вариант развертывания предназначен для одновременного запуска клиентского приложения, серверной части и всех необходимых инфраструктурных сервисов. Пользователь должен изменять только файл `.env`, внимательно проверяя сетевые параметры, значения портов, настройки по умолчанию, имена локальных моделей и иные параметры окружения. Файл `docker-compose.yml` редактировать не требуется. Для корректной работы локального распознавания речи модели Vosk должны быть заранее размещены в директории `./Models/Vosk` относительно каталога, содержащего файлы развертывания. Запуск системы выполняется командой:

```bash
docker compose up -d
```

При использовании локальных моделей Ollama их загрузка в контейнерную среду выполняется отдельной командой:

```bash
docker compose --profile init up ollama-init
```
